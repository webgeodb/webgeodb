## 二、测试失败分析

### 1. EngineRegistry 测试（38/38 失败 → 已修复）

**原因**：
```typescript
// ❌ 错误：EngineRegistry 是单例实例，不是类
import { EngineRegistry } from '../../src/spatial/engine-registry';
const registry = new EngineRegistry();  // TypeError!

// ✅ 正确：使用 EngineRegistryClass
import { EngineRegistryClass } from '../../src/spatial/engine-registry';
const registry = new EngineRegistryClass();
```

**修复**：重写整个测试文件，使用正确的 API：
- `register()` (不是 `registerEngine()`)
- `unregister()` (不是 `unregisterEngine()`)
- `getEngineNames()` (不是 `listEngines()`)
- `getEnginesInfo()` (不是 `getEngineCapabilities()`)

### 2. Turf.js 单位系统问题（7/40 失败）

**原因**：Turf.js 默认使用**度数**，测试期望**米/千米**

```typescript
// 10x10 度数的多边形面积（接近赤道）
area = 1232921098571.2913  // 平方度

// 测试期望
area = 100  // 平方米
```

**失败的测试**：
1. `should calculate area of polygon` - 期望 100，实际 1.23万亿
2. `should calculate length of line` - 期望 14.14，实际 1568.52
3. `should calculate perimeter of polygon` - 期望 40，实际 4430.87
4. `should calculate length of multi-segment line` - 期望 20，实际 2223.90
5. `should calculate distance between points` - 期望 10，实际 1111.95
6. `should calculate distance from point to line` - 参数错误
7. `should handle zero distance` - undefined 错误

**解决方案选项**：
- **选项A**：修改测试，使用小坐标范围（<1度）
- **选项B**：添加单位转换到测试（使用 `@turf/convert-units`）
- **选项C**：修改测试期望值，匹配 Turf.js 默认行为

**推荐**：**选项A** - 使用小坐标范围，保持代码简单
